RÚBRICA DE CUARTA EVALUACIÓN FRONT-END 

Proyecto: Desarrollo de Página Web Inmobiliaria con Front-End y Back-End 

Objetivo: Evaluar el desarrollo de una aplicación web inmobiliaria que permita la gestión de  propiedades, propietarios, fotografías, filtros de búsqueda, visualización detallada y  persistencia de información mediante una solución funcional con front-end y back-end. 

**9\. Persistencia de datos y operaciones sobre la Base de Datos — 10  puntos**

| Criterio  | Puntaje  | Descripción |
| :---- | :---- | :---- |
| Conexión correcta a la Base  de Datos | 2 pts  | El sistema establece   correctamente la conexión  con la base de datos y   permite el acceso a la   información requerida para  el funcionamiento de la  aplicación. |
| Inserción de registros   (CREATE) | 2 pts  | La información ingresada  desde los formularios se  almacena correctamente en  la base de datos,   manteniendo la integridad  de los datos y sus   relaciones. |
| Consulta y recuperación de  registros (READ) | 2 pts  | El sistema obtiene   correctamente la   información almacenada en  la base de datos y la   presenta en los distintos  módulos de la aplicación  según corresponda. |
| Modificación de registros  (UPDATE) | 2 pts  | Las modificaciones  realizadas sobre   propietarios, propiedades,  fotografías u otros registros  se actualizan correctamente  en la base de datos y se  reflejan en la aplicación. |
| Eliminación o cambio de  estado de registros   (DELETE) | 2 pts  | El sistema elimina registros  o modifica correctamente  su estado en la base de  datos, manteniendo el correcto funcionamiento de  la aplicación y la   consistencia de la   información. |
